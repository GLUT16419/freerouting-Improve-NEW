> **版本:** V7 (Urban Traffic Planning Router)

## 一、设计哲学与交通隐喻

### 1.1 核心思想

将 **PCB 自动布线** 视为一个 **微型城市的交通规划问题**：

| 电子领域 | ↔ | 交通领域 |
|:---------|:--:|:---------|
| 器件（封装+DRC间距） | → | 建筑物 |
| 走线（线宽+间距） | → | 火车/汽车轨道 |
| 过孔 | → | 隧道/匝道 |
| 不同信号层 | → | 地面道路·高架路·地铁层 |
| BGA/密集引脚区 | → | 交通枢纽 |
| 网络集群 | → | 城市片区 |
| 拥塞区域 | → | 交通拥堵点 |
| 布线完成 | → | 城市路网竣工 |

### 1.2 四阶段建设顺序

```
传统城市规划流程                    UTPR 布线流程
─────────────────                  ─────────────────
① 城市总体规划                     Phase 0: 城市规划
② 主干道+地铁建设                  Phase 1: 骨干路网
③ 城区路网填充                     Phase 2: 城区交通填充
④ 拥堵路口改造                     Phase 3: 路口精确调度
```

### 1.3 核心创新点

1. **预约制替代事后协商**：时序A\* + STOM（4D时空占用图）让网络在搜索时就预见到未来占用，从源头消除冲突
2. **分层预计算加速**：CRP（Customizable Route Planning）多级分区 + CH（Contraction Hierarchies）收缩层次路网
3. **真正独立的城区并行**：CRP 边界预计算使每个城区可完全独立并行求解
4. **零训练依赖拥塞估计**：FLUTE-MST + 高斯概率扩散，确定性算法替代 ML
5. **增量式冲突修复**：D\* Lite 思想局部重布，避免全盘重来
6. **精确数学兜底**：SAT/ILP 精确求解极少数顽固冲突点
7. **网络智能识别**：从网络名+空间分布自动检测差分对、等长组、高速/模拟信号，无需用户手动配置
8. **智能降级布线**：优先按差分/等长严格约束布线，不满足时自动降低要求确保可布通率
9. **低要求网络放松**：对 GPIO、控制信号、测试点等无严格约束的网络，在不违反 DRC 的前提下放松走线限制，利用空隙空间完成布线，充当中高优先级网络的"柔性填充"
10. **优先级驱动调度**：CRITICAL→HIGH→MEDIUM→LOW→POWER_GND→低要求，逐级利用剩余空间，低要求网作为可牺牲的填充物

## 三、Phase 0：城市规划

### 3.1 CH 分层路网 (Contraction Hierarchies)

**目的：** 构建一个带捷径的层次化路由图，使长距离搜索自动走"高速层"。

**算法步骤：**

```
1. 网格化版图
   gridCols = ceil(boardWidth / cellSize)
   gridRows = ceil(boardHeight / cellSize)
   nodeId = layer * gridRows * gridCols + row * gridCols + col

2. 创建原始边
   - 同层：4-邻域连接，权重 = cellSize
   - 跨层：同位置邻层连接，权重 = VIA_BASE_COST (10.0)

3. 计算节点重要性
   importance = (1 - centralityScore) × 50 + (1 - degreeScore) × 30
   - 版图中心 → 高重要性
   - 版图边缘 → 低重要性
   - 高连接度 → 高重要性

4. 迭代收缩
   for node in sorted(nodes, by=importance asc):
       for each neighbour pair (u, v) of node:
           if witnessSearch(u, v, node) == NEEDS_SHORTCUT:
               addShortcut(u, v, weight=dist(u,node)+dist(node,v))
       remove node from graph
       node.level = batch_level
```

**数据结构：**

```
CHNode:
  int id, layer, gridX, gridY
  double floatX, floatY
  int importance, level
  Map<Integer, Double> originalEdges    // 原始边 (neighbor → weight)
  Map<Integer, ShortcutInfo> shortcuts  // 捷径边

ShortcutInfo:
  int viaNode                // 被绕过的收缩节点
  double weight              // 捷径总权重
  int[] bypassedNodes        // 被捷径跳过的节点链
```

**查询算法 — 双向 Dijkstra：**

```
forwardPQ 从 source 出发，只走 level(target) >= level(current) 的边（向上）
backwardPQ 从 target 出发，只走 level(target) <= level(current) 的边（向下）
在中间汇合时找到最短路径，复杂度 O(√n · log n)
```

### 3.2 概率拥塞估计 (ProbabilisticCongestionEstimator)

**目的：** 在不运行实际 A\* 的情况下，用统计方法预测全板的拥塞分布。

**算法步骤：**

```
1. 对每个网络计算 FLUTE 最小斯坦纳树 (MST)
   - 使用 Prim 算法在引脚点集上构建曼哈顿 MST
   
2. 对每条 MST 边应用高斯模糊扩散
   gaussianKernel1D[i] = exp(-i² / 2σ²)  /  Σexp(-j² / 2σ²)
   - σ = 2.0 个网格单元
   - 核半径 = ceil(3σ) = 6 个单元
   - 每条边上的每个网格单元向其邻域扩散概率
   
3. 所有网络叠加 → 概率需求矩阵 probabilityDemand[row][col]

4. 拥塞概率 = probabilityDemand / capacity
   - capacity = tracksPerCell × signalLayerCount × (1 - obstacleFraction)
```

**输出：** 每个网格单元的拥塞概率 [0, 1] 和梯度向量 (用于绕行引导)。

### 3.3 主干道走廊规划 (MainCorridorPlanner)

**目的：** 识别关键信号（DDR、PCIe、时钟）的预期路径，预留走廊保护。

**算法步骤：**

```
1. 对关键网络计算 FLUTE MST
2. 沿每条 MST 边标记 CORRIDOR_HALF_WIDTH (3) 个单元的走廊区
3. 走廊区优先级 = basePriority × (1 - dist / (halfWidth + 1))
4. 走廊区的路由成本乘以 multiplier，鼓励沿走廊走线
   corridorMultiplier = max(0.1, 1.0 - priority × 0.7)
```

### 3.4 CRP 多级分区 (MultiLevelPartitioner)

**目的：** 将版图划分为 3 级层次结构，实现独立并行布线。

**三级体系：**

```
Level 1: 整个 PCB 版图 (全市)
Level 2: 功能区 (FPGA区、DDR区、电源区、连接器区)
Level 3: 城区 (谱聚类网络集群)
```

**Level 2 — 功能区划分：**

```
1. 将版图划分为粗网格 (如 4×4)
2. 统计每格引脚密度
3. 用贪心洪水填充合并高密度格 → 功能区
4. 网络根据其引脚中心归属到对应功能区
```

**Level 3 — 城区划分：**

```
1. 在每个功能区内，使用 SpectralClusterer 对网络谱聚类
2. 每个聚类 → 一个城区 (District)
3. 城区 BBox = 聚类内所有引脚中心的扩展包围盒
4. 每个城区存储：CH 节点 ID 集、网络编号集、边界点列表
```

### 3.5 边界路径表 (DistrictBoundaryPathTable)

**目的：** 预计算区内和跨区边界点间的最短路径，实现真正的独立并行。

**区内路径：** 同一城区内所有边界点对之间的最短路径（CH 双向 Dijkstra 查询）

**跨区路径：** 同一功能区内不同城区之间，面对面的边界点对之间的最短路径

**编码：**
```
key = (zoneId << 40) | (bpId1 << 20) | bpId2
zoneId > 0  → 城区 ID
zoneId < 0  → 功能区 ID (跨区路径)
```

### 3.6 自动网络智能分析 (AutomaticNetworkAnalyzer)

**目的：** 在没有用户手动配置的情况下，自动识别所有网络的类型、配对关系和分组约束，为后续所有阶段提供网络分类元数据。

#### 3.6.1 网络名解析引擎 (NetNameParser)

**正则匹配规则（按优先级从高到低）：**

| 检测类型 | 匹配模式（大小写不敏感） | 置信度 | 示例 |
|:---------|:------------------------|:------:|:-----|
| **差分对-正端** | `_(P|POS|PLUS|+)$`、`_P\b` | 0.95 | CLK_P, DDR_DQ0_P, USB_DP |
| **差分对-负端** | `_(N|NEG|MINUS|-)$`、`_N\b` | 0.95 | CLK_N, DDR_DQ0_N, USB_DN |
| **差分对-完整名** | `_(DP|DN|P|N)` — 同前缀配对 | 0.90 | ETH_TXP/TXN |
| **时钟信号** | `CLK|CLOCK|REFCLK|SYSCLK` | 0.90 | SYS_CLK, REFCLK_P |
| **复位/使能** | `RESET|RST|ENABLE|EN\b` | 0.70 | RESET_N |
| **DDR 数据线** | `DQ\d+\b|DQS\d*\b` | 0.85 | DDR_DQ0, DQS |
| **DDR 地址/命令** | `ADDR|A\d+\b|BA\d+|CS\b|WE\b|CAS|RAS` | 0.80 | ADDR15, BA0 |
| **PCIe 差分** | `PCIe?_(TX|RX)[PN]`、`PCIE_` | 0.95 | PCIE_TXP, PCIE_RXN |
| **以太网** | `(ETH|RGMII|MII|MDI)[_PN]?\d*` | 0.80 | ETH_TX_P, RGMII_RX_CTL |
| **USB** | `USB[_DPN]?\d*` | 0.85 | USB_DP, USB_DN |
| **HDMI/显示** | `HDMI|TMDS|DVI` | 0.85 | HDMI_TX2P, TMDS_CLK |
| **等长总线前缀** | `^([A-Z]+_?)(\d+)$` — 前缀相同、编号连续的网络 | 0.75 | DATA0..DATA31, ADDR0..ADDR15 |
| **模拟信号** | `VIN|VOUT|ADC|DAC|AMP|SENSE|VBAT|VREF` | 0.65 | ADC_IN0, VREF_P |
| **高速串行** | `SERDES|SATA|SF?FP|LVDS|MIPI` | 0.85 | SERDES_TX_P, LVDS_CLK_N |
| **电源/地** | `^V(CC|DD|SS|EE|PP)\d*|^GND|VREF\b|PWR` | 0.60 | VCC3V3, VDD_CORE, GND |
| **SPI/I2C** | `(SPI|I2C|IIC|SDA|SCL)[_\d]*` | 0.70 | SPI_MOSI, I2C_SCL |

**解析流程：**
```
1. 逐网络扫描，匹配所有正则规则
2. 取最高置信度的匹配作为 primaryClass
3. 记录匹配的分组（如总线编号）
4. 输出：NetClass{type, confidence, groupId, groupIndex}
```

#### 3.6.2 差分对验证器 (DiffPairValidator)

**名字匹配阶段：**
```
扫描所有网络对 (netA, netB):
  1. 去掉层/扇出后缀（如_B1, _VIA），提取核心名
  2. 检查核心名是否构成 P/N、POS/NEG、PLUS/MINUS、+/- 对
  3. 记录候选差分对 (pairCandidate ID, netP, netN, confidence)
```

**空间验证阶段**（引脚位置校验，确认或降级候选）：
```
对每个候选差分对：
  1. 计算 netP 引脚集 PinPosSetP、netN 引脚集 PinPosSetN
  2. 若引脚数不相等 → 置信度 × 0.5（降级为弱候选）
  3. 对所有引脚对按欧氏距离匹配（最近邻匈牙利匹配）：
     - d = distance(pinP_i, pinN_j)
     - 平均匹配距离 avgMatchDist = Σd / n
     - 最大匹配距离 maxMatchDist
  4. 计算空间验证分数：
     spatialScore = 1.0 - min(avgMatchDist / MAX_MATCH_RADIUS(2mm), 1.0)
  5. 最终置信度 = nameConfidence × 0.6 + spatialScore × 0.4
```

**决策规则：**
```
if (finalConfidence ≥ 0.80):  → 强差分对，Phase 1 以 CRITICAL 优先级路由
if (finalConfidence ≥ 0.55):  → 弱差分对，Phase 1 以 HIGH 优先级，宽松耦合
if (finalConfidence < 0.55):  → 不作为差分对，降级为普通单端网络
```

#### 3.6.3 等长组检测器 (LengthMatchGroupDetector)

**检测策略（三阶段）：**

```
阶段 A：总线名检测
  1. 扫描前缀+数字结尾的网络（如 DATA0..DATA31）
  2. 引脚中心点在 Bbox 半径 GROUP_SPATIAL_RADIUS (10mm) 内
  3. 收集 ≥ 4 网组成候选总线组

阶段 B：差分对扩展
  1. 已识别的差分对，其 P/N 两网也应视为等长但不计入组大小

阶段 C：高度对称网络
  1. 特殊前缀：DQ、DQS、ADDR、BA、CS、WE
  2. 同一命名空间的网络自动归为等长组
```

**输出格式：**
```
LengthMatchGroup:
  int groupId
  Set<Integer> netNumbers       // 组成员
  double toleranceFraction      // 初始容差 (0.05 = ±5%)
  double minLength, maxLength   // 参考长度范围
  boolean isDiffPairExtension   // 是否为差分对衍生等长
  int priority                  // 组优先级（DDR > 高速 > 普通）
```

#### 3.6.4 网络元数据输出 (NetClass 数据模型)

```java
NetClass:

### 3.7 区域感知谱聚类 (RegionAwareSpectralClusterer)

**目的：** 替代原有纯空间距离谱聚类，引入网络类型亲和度作为聚类距离度量的一部分，使同类网络（差分对/等长组/总线）更倾向落入同一城区。

#### 3.7.1 网络间距离度量增强

```
原有距离: dist(u,v) = euclidean(center_u, center_v)

增强距离: dist_enhanced(u,v) = α × euclidean(center_u, center_v) 
                                + β × typeDissimilarity(u,v) 
                                + γ × groupAffinity(u,v)

α = 0.50  (空间权重，维持原有的地理邻近性)
β = 0.30  (类型不相似性权重)
γ = 0.20  (分组亲和力权重)
```

**类型不相似度** `typeDissimilarity(u,v)`：
```
if (type(u) == type(v)):    return 0.0    // 同类型→强亲和
if (sameMajorType(u,v)):    return 0.3    // 同大类（如 DDR_DATA 和 DDR_CMD）
if (compatibleType(u,v)):   return 0.6    // 兼容（如 CLOCK 和 DIFF_PAIR）
if (typeConflict(u,v)):     return 1.0    // 冲突（如 ANALOG 和 POWER_GND）
default:                    return 0.5
```

**分组亲和力** `groupAffinity(u,v)`：
```
if (sameDiffPair(u,v)):         return -0.5  // 强吸引→强制同城区
if (sameLengthGroup(u,v)):      return -0.3  // 较强吸引
if (sameBusPrefix(u,v)):        return -0.2  // 同总线
if (crossFunction(u,v)):        return  0.1  // 不同功能区
default:                        return  0.0
```

#### 3.7.2 差分对强制共区策略

```
对每个差分对 (P_net, N_net):
  1. 在谱聚类亲和度矩阵中强制将两网的连接权重设为最大值
  2. 在城区合并阶段检查：若 P/N 被分到不同城区则执行强制合并
  3. 合并后更新城区 BBox 为原两网 BBox 的并集
```

#### 3.7.3 等长组凝聚策略

```
对每个检测到的等长组:
  1. 计算组内所有引脚中心的重心
  2. 以重心为引力中心，在谱聚类亲和度矩阵中施加组内吸引力
  3. 若组内网数 > 8：允许分裂为 2 个子聚类（沿引脚分布长轴）
  4. 分裂后子聚类保持组 ID，在等长约束传递时跨城区协调
```

### 3.8 智能降级管理器 (GracefulDegradationManager)

**目的：** 为每个网络/网络组规划降级路径，优先尝试严格约束布线，不满足时自动降级到宽松约束或普通约束，保证可布通率。

#### 3.8.1 降级级别定义

| 级别 | 名称 | 差分对行为 | 等长组行为 | Phase 1 优先级 |
|:-----|:-----|:-----------|:-----------|:--------------|
| LEVEL_0 | 最优 | 严格耦合（间距=1×线宽，过孔数同步，蛇形线匹配） | 严格容差 (±5%) 同层 | CRITICAL |
| LEVEL_1 | 宽松 | 中等耦合（间距=2×线宽，允许±1过孔差，蛇形线受限） | 中等容差 (±15%) 允许换层 | HIGH |
| LEVEL_2 | 退化 | 弱耦合（间距=3×线宽，独立过孔，不强制等长） | 宽松容差 (±30%) 不强制同层 | MEDIUM |
| LEVEL_3 | 解除 | 作为两个独立单端网布线 | 取消等长约束独立布线 | 按普通网 |
| LEVEL_4 | 放弃 | 标记为不可布，留给人手工 | 标记为不可布 | — |

#### 3.8.2 降级触发条件

```
降级触发器（在 Phase 1/2/3 各阶段结束时检查）:

差分对降级触发器：
  T1: Phase 1 时序 A* 搜索失败（找不到同时满足两条平行轨道的路径）
      → 降级到 LEVEL_1，重试 Phase 1
  T2: Phase 2 城区布线失败（两条网络被冲突区域分隔）
      → 降级到 LEVEL_2，重试 Phase 2
  T3: Phase 3 SAT 求解 UNSAT（因为配对约束导致无解）
      → 降级到 LEVEL_3，重试 SAT
  T4: 若 LEVEL_3 仍失败 → 标记为 LEVEL_4 放弃

等长组降级触发器：
  T1: 组内最长网与最短网长度差 > tolerance × referenceLen
      → 容差翻倍（5%→10%→20%→40%），允许插入蛇形线
  T2: 蛇形线插入空间不足（蛇形区域被其他网络占据）
      → 降级到下一级别
  T3: 组内网络分布在 3+ 个不同城区
      → 允许分裂为子组，各自独立匹配
```

#### 3.8.3 降级反馈回路

```
Phase 1 结束
  → 检查差分对路由状态
  → 若失败 → 降级 → 重试 Phase 1（最多 2 轮）
  → 若成功 → 继续

Phase 2 结束  
  → 检查等长组长度匹配 + 差分对耦合质量
  → 若有未达标且未达到降级上限 → 降级 → 重试该城区
  → 否则 → 继续

Phase 3 结束
  → 检查 SAT 求解结果
  → 若 UNSAT 且可降级 → 降级 → 生成新 SAT 候选
  → 否则 → 标记失败
```

#### 3.8.4 降级日志与可解释性

```
每次降级记录：
  DegradationEvent:
    int netNumber
    int groupId
    Level from, to
    String reason                  // "差分对耦合路由失败"
    int phaseId                    // 触发相位

  [降级记录] DDR_DQ_P/N: LEVEL_1→2 (Phase 2 空间不足)
  [降级记录] DATA[0..15]: LEVEL_0→1 (Phase 3 跨城区太长)
  [最终] 布线完成率: 98.7%

### 3.9 低要求网络放松管理器 (LowRequirementRelaxationManager)

**目的：** 识别无严格电气/信号完整性要求的网络（GPIO、控制信号、测试点、备用焊盘等），在**不违反 DRC** 的前提下为其放松走线限制，允许利用高优先级网络留下的空隙空间完成布线，最大化整体布通率。

**交通类比：** 城市中的"自行车道/人行道"——不占用主干道，利用建筑物缝隙、路肩、绿化带中的小径穿行，只要能到达目的地即可，对路线平直度、车速、路面质量无要求。

#### 3.9.1 低要求网络识别

```
在 AutomaticNetworkAnalyzer 的输出基础上，增加低要求标记：

低要求网络判定规则（满足任一即可）:
  1. NetClass.backbonePriority == LOW (1) 且 NetClass.type == NORMAL
     → GPIO、LED、按键、JTAG、测试点等低速数字信号
  2. NetClass.type == POWER_GND (0) 
     → 电源/地网络（无需信号完整性控制）
  3. NetClass.type == UNKNOWN 且 estimatedLength < SHORT_NET_THRESHOLD (10mm)
     → 短距离未识别网络，大概率是辅助/测试信号
  4. 网络名匹配以下模式之一:
     - GPIO\d* | TEST|TP\d* | NC|DNC|RSVD
     - DEBUG|LED\d*|SW\d*|BUTTON|BUZZER
     - JTAG_\w+|TMS|TCK|TDI|TDO|TRST
     - \w*_FB|SENSE|MON|DET|PG\b|PGOOD
  5. 用户手动标记为低要求（预留接口）

非低要求网络（即使优先级低也受保护）:
  - 任何被识别为差分对/等长组/时钟/模拟的网络
  - 任何被用户手动分配了层/阻抗/等长约束的网络
```

#### 3.9.2 放松规则定义

| 约束项 | 正常模式 | 放松模式 | DRC 保护 |
|:-------|:---------|:---------|:---------|
| **转弯惩罚** | BEND_COST (1.0) | 0（不惩罚转弯） | ✓ |
| **层偏好** | 仅偏好 Grade 层 | 任意可用层 | ✓ |
| **过孔限制** | VIA_BASE_COST (10.0) | VIA_BASE_COST × 0.3 | ✓ |
| **拥塞避让** | probCost × 1.0 | probCost × 0.1（几乎无视拥塞） | ✓ |
| **走廊折扣** | corridorBenefit 正常 | 走廊折扣为 0（不抢占走廊） | ✓ |
| **CH 引导** | chGuidance 0.5× | chGuidance 1.0×（不要求高速层） | ✓ |
| **历史代价** | CONFLICT_PENALTY (500) | CONFLICT_PENALTY × 0.1 | ✓ |
| **STOM 预约惩罚** | STOM_PENALTY (100) | STOM_PENALTY × 0.2（允许高占用区穿行） | ✓ |
| **最小线长优先** | 是（倾向短路径） | 否（允许绕远路） | ✓ |
| **蛇形线插入** | 不允许（浪费空间） | 允许（利用剩余通道） | ✓ |
| **层切换成本** | 高 | 低（自由换层找空隙） | ✓ |

**核心原则：** 所有 DRC 规则（线宽、间距、环宽、过孔尺寸等）**严格执行**，放松的仅是"走线质量"相关的代价项。

#### 3.9.3 阶段调度 — 最后布线，利用空隙

```
布线时序（按 Phase 2 城区内排序）:
  1. 地铁层网络 (Subway)       ← 高速/差分，走专用层对
  2. 高架层网络 (Elevated)     ← 中速/模拟，走中间层对
  3. 地面层网络 (Surface)      ← 普通低速，走表层
  4. 低要求网络 (Relaxed)      ← 在以上三层都已完成后的空隙中穿行

低要求网络布线入口：
  Phase 1: 不参与骨干网选择（跳过 Phase 1）
  Phase 2: 在本城区所有正常网络完成后，最后执行
  Phase 2.5: 在 IncrementalRerouter 修复冲突前，先尝试为未布的低要求网找路
  Phase 3: 若仍未布通，在 SAT 求解阶段移除所有约束，仅保留 DRC
  Final Cleanup: 作为 ripup 兜底的首选牺牲品（ripup cost = 0.1）
```

#### 3.9.4 缝隙填充走线策略

```
对每个低要求网络，在正常网络布线完成后执行"缝隙填充走线"：

1. 扫描 STOM 当前占用情况，建立空闲通道图：
   - 将 STOM 中 occupancyCount == 0 的单元格标记为"空闲"
   - 将所有 obstacles 标记为"不可用"
   
2. 使用简化 A* 搜索（关闭质量项）:
   ```
   g(n) = moveCost × 1.0 + stomCost × 0.2 + viaCost × 0.3 + probCost × 0.1
   ```
   - stomCost 极低：容忍高占用区域穿行
   - viaCost 极低：允许通过过孔找空隙
   - probCost 极低：几乎无视拥塞热力图
   - 无 bendCost：不惩罚转弯
   - 无 corridorBenefit：不进入走廊保护区
   - 无 chGuidance：不要求 CH 高速层

3. 路径评估：
   - 只要找到一条合法路径（无 DRC 冲突）即接受
   - 不比较路径长度、过孔数、转弯数等质量指标
   - 搜索超时: LOW_REQ_TIMEOUT (3s) — 比正常网络短

4. 若搜索失败：
   - 回退到 routeNet() 标准搜索（但保持放松代价系数）
   - 若仍然失败 → 触发清理器（Cleanup Pass）中的最终尝试
```

#### 3.9.5 放松与非放松网络的交互约束

```
1. 已布线的正常网络 → 视为障碍物（DRC 间距保护），不允许低要求网贴边挤压
2. 低要求网络之间 → 允许更紧密排列（但仍遵守 DRC 间距）
3. 低要求网络走线后 → 不在 STOM 中预约时间步（不影响后续正常网重路由）
4. 低要求网络走线后 → 若后续正常网需要重布，低要求网可被 ripup（ripup cost = 0.1）
5. 电源/地网络放松 → 允许走任意层，但建议保持铜皮完整性（特殊处理）

事后清理机制:
  - 若低要求网络占据的空间导致后续并行布线冲突
  - IncrementalRerouter 优先 ripup 低要求网释放空间
  - 被 ripup 的低要求网进入 Final Cleanup 阶段重新布线
```

### 4.1 STOM 时空占用图 (SpatioTemporalOccupancyMap)

**核心数据结构：** 4D 稀疏时空占用图。

```
STOM[x][y][layer][timeStep]
  x, y      = 网格坐标
  layer     = 信号层
  timeStep  = 逻辑时间步（布线顺序编号）
```

**实现：**
```
Map<Long, BitSet> occupancyMap  // cellKey → 该单元格各时间步的占用位图
Map<Integer, Set<Long>> reservationLog  // timeStep → 该时间步占用的单元格集

cellKey = (layer << 40) | (row << 20) | col  // 3→1 编码
```

**核心操作：**

| 操作 | 描述 | 复杂度 |
|:-----|:-----|:-------|
| `reserve(path)` | 分配一个新时间步，标记路径上所有单元格 | O(pathLen) |
| `query(x,y,layer,t)` | 检查某单元格在时间步 t 是否被占用 | O(1) |
| `release(t)` | 释放时间步 t 的所有预约 | O(occupiedCells) |
| `getOccupancyCount(x,y,layer)` | 获取某单元格的总预约次数 | O(1) |

### 4.2 骨干网络选择 (BackboneNetSelector)

**优先级体系（自动与用户配置合并）：**

| 优先级 | 值 | 适用网络 | 自动识别来源 |
|:-------|:--:|:---------|:------------|
| CRITICAL | 4 | 差分对（强）、高速时钟、DDR、PCIe、SERDES | AutomaticNetworkAnalyzer |
| HIGH | 3 | 差分对（弱）、等长组、模拟、ADC/DAC、HDMI、USB | AutomaticNetworkAnalyzer |
| MEDIUM | 2 | 长距离普通数字、SPI/I2C、组内网络 | 自动回退 |
| LOW | 1 | 短距离普通数字、低速控制信号 | 自动回退 |
| POWER_GND | 0 | 电源/地、VREF | AutomaticNetworkAnalyzer |

**自动优先级分配规则：**
```
1. 若用户已手动配置了网络的优先级 → 以用户配置为准（跳过自动）
2. 否则查询 AutomaticNetworkAnalyzer 的 NetClass：
   - NetClass.type == DIFF_PAIR && confidence ≥ 0.80 → CRITICAL
   - NetClass.type == DIFF_PAIR_WEAK                  → HIGH
   - NetClass.type == CLOCK || DDR_DATA || DDR_CMD     → CRITICAL
   - NetClass.type == LENGTH_MATCH_GROUP                → HIGH
   - NetClass.type == ANALOG || ETHERNET || USB || HDMI → HIGH
   - NetClass.type == I2C_SPI                           → MEDIUM
   - NetClass.type == POWER_GND                         → POWER_GND
   - NetClass.type == NORMAL || UNKNOWN:
        estimatedLength > LONG_NET_THRESHOLD → MEDIUM
        否则 → LOW
3. 差分对的两条网络自动获取相同优先级（保证同时调度）
4. 等长组的所有成员自动获取组内最高优先级
```

**排序规则：** `(priority DESC) → (channelScore DESC) → (estimatedLength DESC) → (diffPairId != -1 DESC)`

### 4.4 时序双向 A\* (TimedBidirectionalAStar)

**代价函数：**

```
// 标准代价函数（关键/常规网络）
f(n) = g(n) + h(n)

g(n) = moveCost + stomCost + probCost + bendCost + corridorBenefit + chGuidance

moveCost = CELL_SIZE × corridorMultiplier       // 基础移动 × 走廊折扣
stomCost = STOM_PENALTY × occupancyCount         // 已有预约惩罚 (100×)
probCost = congestion × 10.0                     // 概率拥塞惩罚
bendCost = turning ? BEND_COST : 0               // 转弯惩罚 (1.0)
corridorBenefit = corridorMultiplier < 1 ? yes   // 走廊优惠
chGuidance    = on CH path ? 0.5× : 1.0×         // CH 路径引导

h(n) = manhattan(n, target) × HEURISTIC_WEIGHT
```

**放松代价函数（低要求网络，由 LowRequirementRelaxationManager 切换）：**

```
// 低要求网络使用放松代价
g_relaxed(n) = moveCost                                               
                + stomCost × 0.2          // 无视已有预约
                + probCost × 0.1          // 几乎无视拥塞
                + viaCost × 0.3           // 过孔代价值极低
                + 0                       // 无转弯惩罚
                + 0                       // 无走廊折扣（不抢占走廊）
                + chGuidance × 1.0        // 不要求 CH 高速层

// 启发式不变
h(n) = manhattan(n, target) × HEURISTIC_WEIGHT  
```

**搜索流程：**

```
1. 计算曼哈顿启发式 h
2. 使用 CH 查询获取粗略路径引导 (chPathGuide)
3. 双向 A* 扩展：
   正向：从 source 向外扩展
   反向：从 target 向外扩展
   扩展时：检查 4-邻域 + 层切换（过孔）
4. 在中间节点汇合 → 构建完整路径
5. 调用 stom.reserve(path) 预约路径
```

## 五、Phase 2：城区交通填充

### 5.1 交通分流层分配 (TrafficModeLayerAssigner)

**三层交通体系：**

| 模式 | 对应层对 Grade | 适用场景 | 自动识别类型 |
|:-----|:--------------:|:---------|:------------|
| Surface (地面层) | GRADE_SURFACE (0) | 短距离低速连接、表层微带线 | I2C/SPI、低速控制、LOW 优先级 |
| Elevated (高架层) | GRADE_ELEVATED (1) | 中距离、模拟信号 | ANALOG、ADC/DAC、HIGH 优先级普通网 |
| Subway (地铁层) | GRADE_SUBWAY (2) | 长距离高速、差分对、DDR | DIFF_PAIR、CLOCK、DDR、SERDES、MEDIUM+ |

**自动层偏好分配（基于 AutomaticNetworkAnalyzer 输出）：**

```
对每个网络：
  1. 查询 NetClass.preferredLayerGrade:
     - DIFF_PAIR(强)    → preferredGrade = 2 (Subway)
     - DIFF_PAIR_WEAK   → preferredGrade = 1 (Elevated)
     - CLOCK/DDR/PCIe   → preferredGrade = 2 (Subway)
     - ANALOG           → preferredGrade = 1 (Elevated)
     - POWER_GND        → preferredGrade = 0 (Surface)
     - BUS_GROUPED      → 组内统一分配（全部 Subway 或全部 Elevate）
     - NORMAL/UNKNOWN:
         if (estimatedLength > LONG_NET_THRESHOLD) → preferredGrade = 2
         else → preferredGrade = 0

  2. 若用户手动指定了层/层对 → 以用户配置为准

  3. 差分对强制规则：
     - 同一差分对的两网必须分配到同层对
     - 若该层对容量不足，触发 GracefulDegradationManager 降级

  4. 等长组强制规则：
     - 等长组内所有网络分配到同一 Grade 层对（或兼容层对）
     - 层对差异过大致蛇形线长度超过限制 → 触发降级
```

**软分配机制：**

```
1. 分析网络属性：长度、速度、信号类型（来自 NetClass）
2. 确定偏好模式（preferredMode = preferredGrade）
3. 从 LayerPair 列表中选择最优匹配
4. 通过修改各层代价系数实现软偏好：
   - 偏好层走线成本低，非偏好层走线成本高
   - 层接近饱和时自动提高成本，引导其他网络改道
```

**容量监控：**
```
modeRemainingCapacity[mode] -= newlyOccupiedArea
if (modeRemainingCapacity[mode] < threshold):
    modeCostMultiplier[mode] *= 1.5  // 抬价引导其他网绕行
if (modeRemainingCapacity[mode] < CRITICAL_THRESHOLD && networkType == DIFF_PAIR):
    // 通知 GracefulDegradationManager 该层对无法容纳差分对
    degradationManager.trigger(DIFF_PAIR_NO_CAPACITY, netNumber)
```

### 5.2 城区并行布线 (DistrictRouter)

**设计：**

```
线程池并行调度所有城区
每个城区内部：
  1. 获取城区网络列表，按 isLowRequirement 分为"正常组"和"低要求组"
  2. 正常组按交通模式排序（地铁 → 高架 → 地面）
  3. 依次调用 BatchAutorouter.routeNet() 路由正常组
  4. 低要求组路由（最后，利用空隙空间）:
     a. 调用 LowRequirementRelaxationManager.applyRelaxedCosts()
     b. 对每个低要求网使用放松代价搜索
     c. routeNet() 使用放松代价函数
     d. 成功后在 STOM 中标记占用（允许被后续 ripup 回收）
  5. 所有网络完成后 → 进入跨区对接
```

**并行度控制：**
```
threadCount = min(MAX_DISTRICT_THREADS, min(districtCount, availableProcessors))
每个城区超时限制 2 分钟
```

### 5.3 跨区交通对接 (CrossDistrictConnector)

**处理跨城区网络：**

```
1. 识别跨区网络（引脚分布在两个以上城区）
2. 对每对跨区端点：
   a. 从出发城区边界点出发
   b. 通过预计算跨区路径表找到对接路径
   c. 在目标城区边界点接入
3. 拼接完整路径并布线
4. 回退方案：使用标准 BatchAutorouter.routeNet()
```

### 5.4 D\* Lite 增量重布 (IncrementalRerouter)

**算法流程：**

```
0. 低要求网络优先 ripup（释放空间给正常网络）
   - 扫描所有占用单元格，优先释放 isLowRequirement == true 的网络
   - 被 ripup 的低要求网暂存至待布线列表（最后重新布线）
   - ripup cost = 0.1（几乎是免费的）

1. 冲突检测
   - 扫描 STOM 中 occupancyCount > 1 的单元格
   - 对冲突单元格做 4-邻域洪水填充 → 冲突区域

2. 历史代价注入
   - 冲突区域每个单元格增加 CONFLICT_PENALTY (500)
   - 后续搜索自动避开这些区域

3. 增量重路由（正常网络）
   - 对冲突区域中涉及的 2-5 条非低要求网络
   - 以高 ripup cost (10) 调用 routeNet()
   - routeNet 的高历史代价会引导路径绕行

4. 低要求网络重布线
   - 在冲突解决后，对步骤 0 中被 ripup 的低要求网重新布线
   - 使用放松代价函数（参见 3.9.4）
   - 若仍无法布通 → 进入 Final Cleanup Passes

5. 迭代收敛
   - 重复 1-4 直到冲突区域 ≤ 10 个
   - 停滞检测：3 轮无改善则停止
   - 剩余冲突移交 Phase 3
```

## 六、Phase 3：路口精确调度

### 6.1 瓶颈路口识别 (BottleneckAnalyzer)

**空间聚类算法：**

```
1. 收集所有剩余未布线网络
2. 提取各网络的引脚重心
3. 基于 CLUSTER_RADIUS (5mm) 做贪心空间聚类
4. 计算每个聚类的密度分数：
   densityScore = conflictCount / area
5. 按密度降序排列
6. 将 > 8 网络的超大聚类沿长轴中位数分裂
```

**输出：**
```
BottleneckRegion:
  int id
  double minX, minY, maxX, maxY  // 紧致包围盒
  Set<Integer> involvedNetNumbers // 涉及的网络
  int conflictCount               // 冲突数
  double densityScore             // 密度分数
```

### 6.2 候选路径生成 (BottleneckSatSolver)

**多样候选路径：**

```
对每个瓶颈网络：
  1. 运行 MAX_CANDIDATES_PER_NET (8) 次带随机扰动的 routeNet()
  2. 扰动参数：
     - ripupCost = BASE_RIPUP_COST + relaxation × 2 + i
     - 随机种子 = random.nextLong()
  3. 记录每条路径的占用单元格哈希
  4. 按总成本排序，保留最优的 8 条
```

### 6.3 SAT/ILP 求解

**编码方式（复用 Sat4jSolver）：**

```
变量： 每条候选路径对应一个布尔变量
硬约束：
  - 每个网络必须选且仅选一条路径 (at-least-one + at-most-one)
  - 冲突路径互斥（占用相同网格的路径不能同时选）
  - 差分对必须同步选择配对路径
软约束：
  - 最小化总路径长度
  - 最小化过孔数
  - 等长组内长度差异 ≤ 20%
```

**并行求解：**

```
4 个并行变体，不同松弛级别
超时 30 秒
最优结果 = 最多解决网络数 → 最低总成本
```

### 6.4 UNSAT 恢复与局部解冻

**UNSAT 恢复流程：**

```
if (parallelSolve returns UNSAT):
    1. 提取 UNSAT 核心（未分配的网络）
    2. 为 UNSAT 核心生成额外候选路径（更高 ripup cost）
    3. 重试 SAT 求解（最多 5 轮，每轮增加松弛级别）
if (still UNSAT):
    4. 局部解冻：解冻瓶颈周围 MAX_THAW_NETS (4) 条非关键网络
    5. 重新生成候选并求解
if (still UNSAT):
    6. 低要求网络松绑：将 UNSAT 核心中的低要求网络标记为
       "无 SAT 约束"，允许路由到任意空闲通道（不受长度/层限制）
    7. 重新生成候选并求解
if (still UNSAT):
    8. 标记为最终失败（极小概率）
```

### 6.5 Final Cleanup Passes（最终清理贯通）

**目的：** Phase 3 完成后，对所有尚未布线（包括被 ripup 的低要求网）做最后一次兜底尝试。

```
1. 收集所有未布网络（包括被 ripup 的低要求网）

2. 按优先级分组：
   组 A: 关键/高优先级未布网络（CRITICAL / HIGH）
         → 使用标准 routeNet() + 全部历史代价 + 长超时
   组 B: 中/低优先级未布网络（MEDIUM / LOW）
         → 使用 routeNet() + 中等历史代价 + 标准超时
   组 C: 低要求放松网络（isLowRequirement == true）
         → 使用简化缝隙填充（参见 3.9.4）+ 短超时
   组 D: 电源/地
         → 使用标准 routeNet() + 允许任意层

3. 多轮次尝试（最多 3 轮）：
   轮 1: 所有组顺序尝试（A→B→C→D）
   轮 2: 仅剩的未布网，降低所有约束级别
   轮 3: 仅剩的未布网，全部使用低要求放松代价函数

4. 最终统计
   - 成功布线的网络数
   - 因 DRC 无法布线的网络数（留给用户手动处理）
   - 低要求网络布通率（期望 ≥ 99%）
```

### 7.2 自动识别与降级联动约束

**基于自动识别的约束增强流程：**

```
1. AutomaticNetworkAnalyzer 输出 NetClass 元数据

2. GracefulDegradationManager 为每个网络/组设定初始降级级别：
   - 强差分对 (confidence ≥ 0.80) → LEVEL_0
   - 弱差分对 (0.55 ≤ confidence < 0.80) → LEVEL_1
   - 等长组 → LEVEL_0 或 LEVEL_1（根据组大小和空间分布）
   - 普通网络 → 不降级

3. 约束传递规则：
   a. 差分对 → 降级级别决定耦合强度
      LEVEL_0: 间距=1×线宽, 过孔同步, 等长匹配, 同层
      LEVEL_1: 间距=2×线宽, 允许±1过孔差
      LEVEL_2: 间距=3×线宽, 独立过孔
      LEVEL_3: 解除配对, 独立单端路由
   
   b. 等长组 → 降级级别决定容差
      LEVEL_0: ±5%
      LEVEL_1: ±15%
      LEVEL_2: ±30%
      LEVEL_3: 移除等长约束
   
   c. 层分配约束 → 降级级别决定是否允许换层
      LEVEL_0: 仅偏好层
      LEVEL_1-2: 允许相邻 Grade 层
      LEVEL_3: 允许任意层

4. 降级触发条件（自动检测）：
   - Phase 1 失败的差分对 → 降一级
   - Phase 2 长度不匹配的等长组 → 降一级
   - Phase 3 SAT UNSAT 的配对约束 → 降一级
   - 连续 2 轮降级后仍失败 → 跳级（LEVEL_0 → LEVEL_2）
```

## 附录 A：算法复杂度

| 组件 | 时间复杂度 | 空间复杂度 |
|:-----|:----------:|:----------:|

### C.2 STOM 并行隔离/合并 (Parallel Copy/Merge/Isolate)

**原状态**：STOM 仅支持串行预约，城区并行布线时无法创建独立的 STOM 副本。

### C.3 STOM 时间步分配策略 (Strategic Time-Step Allocation)

**原状态**：`reserve()` 简单递增 timeStep 计数器，无策略。

**修复**：新增 `allocateTimeStep(cellKeys, priority, reservedSteps)` 方法，支持三种策略：

### C.4 双向 A\* 真双向逻辑 (True Bidirectional A\*)

**原状态**：`TimedBidirectionalAStar.search()` 的终止条件仅为 `f >= bestPathCost`，缺少两方向最小 f 值之和判断；会陷入单向扩展。

**修复**：实现真正双向 A\* 终止：
- 每次循环计算 `minFwdF = openFwd.peek().f` 和 `minBwdF = openBwd.peek().f`
- 强终止条件：`minFwdF + minBwdF >= bestPathCost` 时提前退出（保证最优性）
- 平衡扩展策略：选择 f 值较小的方向优先扩展
- 使用 parent maps（`parentFwd`/`parentBwd`）替代 SearchNode 链重建路径

### C.5 差分对联动预约 (Differential Pair Interlocked Routing)

**修复**：在 `expandNode()` 中增加了差分对间距代价逻辑：

- 如果在 `DIFF_PAIR_SPACING` (3 格) 内有同伴网络占用 → 给予负代价（奖励保持间距）

### C.6 等长组协同规划 (Matched-Length Group Coordination)

**修复**：完全重构 `BackboneNetSelector.java`：

  3. 差分对 `_P`/`_N` 配对

### C.7 跨区对接 4 步流程 (Cross-District 4-Step Connection)

**原状态**：`CrossDistrictConnector.identifyCrossDistrictNets()` 返回空集合，`connectNet()` 使用 `routeNet` 降级。

**修复**：完全重写 `CrossDistrictConnector.java`，实现完整 4 步流程：
1. **Step 1 (识别跨区网)** — 为每个网检查所有引脚所在城区，跨越 ≥2 个区则为跨区网
2. **Step 2 (确定区序列)** — 构建城区邻接图（BBox 重叠/邻近判定），BFS 求源→目标区最短路径
3. **Step 3 (匹配边界点)** — 遍历相邻城区对的 boundaryPoints，欧氏距离最近配对

### C.8 D\* Lite 真实增量算法 (True D\* Lite)

**原状态**：`IncrementalRerouter` 仅实现了 "检测冲突→ripup→重布" 的 PathFinder 风格流程，无 g/rhs 值、无 DKey 队列、无 ComputeShortestPath。

**修复**：完全重写 `IncrementalRerouter.java`，实现完整 D\* Lite 算法：
- **DKey** — 优先队列键 `[min(g,rhs) + h, min(g,rhs)]`
- **DNode** — 每个节点维护 `g` / `rhs` 值
- **ComputeShortestPath** — 当 `U.TopKey() < CalculateKey(goal) || rhs(goal) != g(goal)` 时循环
- **UpdateNode(u)** — 重算 u.rhs = min(c(u,v) + g(v))，同步更新队列
- **Over-consistent / Under-consistent** 分支处理
- 历史冲突代价叠加：冲突区细胞获得额外成本增量，使 D\* Lite 自动规划绕行
