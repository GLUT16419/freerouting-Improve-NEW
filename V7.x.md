# UTPR V7.x — 城市交通规划布线引擎 V7.x 深化优化文档

> **版本:** V7.3 (基于 V7 UTPR 框架的 8 项优化)  
> **发布日期:** 2026-07-02  
> **基础:** V7 UrbanTrafficBatchAutorouter  
> **关键词:** CH层感知拆分，预判性降级，历史代价冷却衰减，等长组扩散增强，拥塞梯度代价，时间窗软化，最大空隙优先，4档速度预设

## 一、概述

V7.x 是 V7 UrbanTrafficBatchAutorouter 的深化优化版本，将地图导航算法思想进一步映射到 PCB 布线中。共实施 **8 项优化**，分为效率、精度、可扩展性三个维度，全部通过配置开关控制，向后兼容。

### 1.1 核心创新

1. **CH 层感知拆分** — 按 Subway/Elevated/Surface 三交通模式拆分 CH 子图，搜索时仅使用目标层 CH 节点，缩小搜索图 40-60%
2. **预判性降级** — 利用 Phase 0 拥塞热力图预判差分对冲突级别，跳过无效 LEVEL_0 尝试，减少无效重试 20-30%
3. **历史代价冷却衰减** — 被释放网格的历史代价按 x0.5 指数衰减，避免过度保守，提高通道复用率
4. **等长组扩散增强** — 以组为单位扩散外包盒，乘以 1.0+log2(size)×0.3 因子预占蛇形线空间，拥塞估计更准
5. **拥塞梯度代价** — 扩展时参考拥塞梯度方向，沿梯度惩罚、逆梯度奖励，减少撞墙式绕行
6. **时间窗软化** — 允许时间差 <= 3 步的网络共享网格（软共享代价 10），通道利用率提升
7. **最大空隙优先** — 低要求网络搜索前用二维前缀和定位最大空闲矩形，引导 A* 沿空隙走，布通率+10-20%
8. **4 档速度预设** — VERY_FAST/FAST/MEDIUM/SLOW 四档灵活控制速度与质量平衡

## 三、每个模块的设计

### 3.1 CH 层感知拆分

**文件:** `ContractionHierarchies.java` + `UrbanTrafficRouterSettings.chLayerAware`

**设计:**
- `contract()` 按交通模式拆分 CH 子图
- `query()` 支持 `layerMask` 参数（LAYER_MASK_ALL/SUBWAY/ELEVATED/SURFACE）
- 交通模式到层对的映射: Subway=Grade 2(内层), Elevated=Grade 1(中间层), Surface=Grade 0(表层)
- `isLayerExcluded()` 按 Grade 过滤节点

### 3.2 预判性降级

**文件:** `GracefulDegradationManager.java`

**设计:**
- `buildPredictivePlan(double[][] congestion)` 在 Phase 0 拥塞估计完成后调用
- 对每个差分对/等长组估算必经区域加权拥塞指数
- weightedCongestion > 0.85 → LEVEL_1 启动（跳过 LEVEL_0 无效尝试）
- weightedCongestion > 0.70 → Phase 1 超时减半

### 3.3 历史代价冷却衰减

**文件:** `IncrementalRerouter.java`

**设计:**
- `decayHistoricalCosts()` 扫描 `historicalConflictCost` map
- 对 occupancyCount 已降为 0 的网格，代价乘以 coolingFactor
- 低于 MIN_HISTORICAL_COST (1.0) 时移除 → 完全恢复

### 3.4 等长组扩散增强

**文件:** `ProbabilisticCongestionEstimator.java`

**设计:**
- `diffuseLengthGroups()` 对每个等长组计算联合外包盒
- 强度 = `1.0 + log2(groupSize) × 0.3`（上限 3×）

### 3.5 拥塞梯度代价

**文件:** `TimedBidirectionalAStar.java`

**设计:**
- 在 `expandNode()` 中获取拥塞梯度向量
- 计算扩展方向与梯度方向的点积
- `gradientCost = max(0, dotProduct) × congestionGradientWeight`

### 3.6 时间窗软化

**文件:** `SpatioTemporalOccupancyMap.java`

**设计:**
- `querySoft()` 返回冲突级别: 0=空闲, 1=软冲突, 2=硬冲突
- 查询范围 `[timeStep - windowSize, timeStep + windowSize]`
- `getSoftSharingCost()` 映射冲突级别为代价

### 3.7 最大空隙优先

**文件:** `MaxGapFinder.java` (新增) + `LowRequirementRelaxationManager.java`

**设计:**
- 二维前缀和矩阵实现 O(1) 区域空闲度查询
- 贪心扫描从源到目标寻找最大连续空闲矩形
- 长宽比保持在 [0.3, 3.0] 范围内
